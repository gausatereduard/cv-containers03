### Запуск и тестирование

1. Создайте в папке containers03 файл Dockerfile со следующим содержимым:

    ```dockerfile
    FROM debian:latest
    COPY ./site/ /var/www/html/
    CMD ["sh", "-c", "echo hello from $HOSTNAME"]
    ```

    В той же папке проекта создайте папку site. В новой папке создайте файл index.html с произвольным содержимым.

2. Откройте терминал в папке containers03 и выполните команду:

    `docker build -t containers03 .`

    ---

    > Сколько времени создавался образ? — 0.6 секунды

3. Выполните команду для запуска контейнера

    `docker run --name containers03 containers03`

    ---

    > Что было выведено в консоли?

    ```bash
    hello from e3aa920055f4
    ```

4. Удалите контейнер и запустите снова, выполнив команды:

    ```bash
    docker rm containers03
    docker run -ti --name containers03 containers03 bash
    ```

5. В открывшемся окне выполните команды:

    ```bash
    cd /var/www/html/
    ls -l
    ```

    > Что выводится на экране?

    ```bash
    root@633e015c3729:/var/www/html# ls -l
    total 4
    -rw-r--r-- 1 root root 252 Feb 15 23:36 index.html
    ```

### Вывод

Контейнеры докера поднимаются явно быстрее, чем с использованием QEMU. Однако в случае с QEMU - устанавливается обычная ОС, тогда как в контейре могут быть запущены как `alpine` версии некоторых ОС, так и обычные, а так же просто готовые приложения по типу `redis`, `postgres`, `rabbitmq`
